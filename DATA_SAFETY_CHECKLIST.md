# Google Play 数据安全快速检查清单

## 必须在Google Play Console中声明的数据

### ✅ 必须声明

1. **设备ID**
   - 类型：设备ID或其他ID → 设备ID
   - 用途：应用功能、分析
   - 共享：是（友盟、RevenueCat）
   - 加密：是

2. **应用使用统计**
   - 类型：应用活动 → 应用交互
   - 用途：分析
   - 共享：是（友盟）
   - 加密：是

3. **购买记录**
   - 类型：财务信息 → 购买历史
   - 用途：应用功能
   - 共享：是（RevenueCat）
   - 加密：是

4. **崩溃日志**（如果友盟收集）
   - 类型：应用信息和性能 → 崩溃日志
   - 用途：分析
   - 共享：是（友盟）
   - 加密：是

### ❌ 不需要声明（仅本地存储）

- 用户创建的事件数据（仅存储在本地设备）
- 用户设置的提醒（仅存储在本地设备）
- 应用内设置（仅存储在本地设备）

## 第三方SDK声明

### 友盟统计（Umeng）
- **必须声明**：是
- **收集数据**：设备ID、应用使用统计、崩溃日志
- **隐私政策**：需要在友盟官网查找

### RevenueCat
- **必须声明**：是
- **收集数据**：购买记录、订阅状态
- **隐私政策**：https://www.revenuecat.com/privacy

## 权限说明

### READ_PHONE_STATE
- **用途**：仅用于获取设备ID（`androidInfo.id`）
- **不收集**：电话号码、IMEI、IMSI
- **说明**：如果Google询问，请说明仅用于设备标识，不收集敏感信息

## 提交前检查

- [ ] 已声明设备ID收集
- [ ] 已声明应用使用统计
- [ ] 已声明购买记录
- [ ] 已声明第三方SDK（友盟、RevenueCat）
- [ ] 已提供隐私政策链接
- [ ] 数据用途说明准确
- [ ] 数据共享情况准确
- [ ] 数据加密方式已说明

## 常见拒绝原因

1. **未声明设备ID收集** - 应用收集了设备ID但未在数据安全部分声明
2. **未声明第三方SDK** - 使用了友盟、RevenueCat等SDK但未声明
3. **权限用途不明确** - READ_PHONE_STATE权限用途需要说明
4. **隐私政策缺失** - 未提供可公开访问的隐私政策链接

## 快速修复步骤

1. 登录 [Google Play Console](https://play.google.com/console)
2. 选择应用：Days Reminder - 倒数日历
3. 进入：**政策** → **数据安全**
4. 按照 `GOOGLE_PLAY_DATA_SAFETY_GUIDE.md` 中的详细指南填写
5. 保存并提交审核
